## GitHub Project Board

The project board should be set up with these columns:

1. **Incoming** - New gigs auto-created by scraper
2. **Reviewing** - Gigs currently being evaluated
3. **Applied** - Gigs Jad has applied to
4. **Rejected** - Gigs passed on (with reasons)

## Issue Templates

### 1. New Gig Found (Auto-Generated)

**Template**: `.github/ISSUE_TEMPLATE/new-gig-found.md`

**Trigger**: When scraper finds a new gig opportunity

**Labels**: `new-gig`, `incoming`

**Required Fields to Populate**:
- `{{platform}}` - e.g., "Upwork", "Fiverr", "Freelancer"
- `{{title}}` - Job/gig title
- `{{company}}` - Client/company name
- `{{location}}` - Remote, on-site, or location
- `{{rate}}` - Pay rate or budget
- `{{posted_date}}` - When it was posted (ISO 8601)
- `{{url}}` - Direct link to the gig
- `{{description}}` - Cleaned job description
- `{{raw_json}}` - Full scraped data as JSON

**Example API Call**:
```bash
gh issue create \
  --repo atEverychance/gig-economy \
  --template "new-gig-found.md" \
  --title "[GIG] Upwork - Senior React Developer" \
  --label "new-gig,incoming"
```

### 2. Gig Review Notes (Manual)

**Template**: `.github/ISSUE_TEMPLATE/gig-review-notes.md`

**Trigger**: When Jad wants to document review notes

**Labels**: `review-notes`

**Usage**: Linked to original gig issue via `original_issue_number`

## GitHub CLI Commands for Scraper

### Create a New Gig Issue

```bash
# Create issue from template
gh issue create \
  --repo atEverychance/gig-economy \
  --template "new-gig-found.md" \
  --title "[GIG] PLATFORM - JOB_TITLE" \
  --label "new-gig,incoming" \
  --body "$(cat populated_template.md)"
```

### Add Issue to Project Column

After creating the issue, add it to the "Incoming" column:

```bash
# Note: This requires project board setup first
gh project item-add --owner atEverychance --project "Gig Review Workflow" --content-id "$ISSUE_NODE_ID"
```

### List Recent Gigs

```bash
gh issue list --repo atEverychance/gig-economy --label "new-gig" --limit 20
```

### Search for Duplicate Gigs

Before creating a new issue, check for duplicates:

```bash
gh search issues "[GIG] PLATFORM - JOB_TITLE" --repo atEverychance/gig-economy
```

## Rate Limiting Considerations

- GitHub API: 5,000 requests/hour for authenticated users
- Consider batching gig discoveries to avoid hitting limits
- Use issue labels to track processing state
